# 教培直播系统 - 部署文档

> **版本**: v1.0.0  
> **适用环境**: CentOS 7+ / Ubuntu 20.04+ / 其他 Linux 发行版

---

## 一、部署方式选择

| 方式 | 适用场景 | 复杂度 | 推荐度 |
|------|---------|--------|--------|
| **PM2 部署** | 单台/多台服务器 | 低 | ⭐⭐⭐⭐⭐ |
| **Docker Compose** | 容器化部署 | 中 | ⭐⭐⭐⭐ |
| **Kubernetes** | 大规模集群 | 高 | ⭐⭐⭐ |

本系统推荐 **PM2 部署**，简单高效，适合大多数场景。


配置 Redis（可选）：

#### 安装 Nginx


## 三、PM2 部署（推荐）

### 3.1 部署步骤

```bash
# 1. 克隆项目
cd /var/www
git clone <your-repo-url> edu-live-server
cd edu-live-server

# 2. 安装依赖
npm ci --only=production

# 3. 配置环境变量
cp .env.example .env
vim .env
```

编辑 `.env` 文件，填入以下关键配置：

```bash
# ===== 基础配置 =====
NODE_ENV=production
PORT=8080
BASE_URL=https://api.yourdomain.com

# ===== MySQL =====
DB_HOST=localhost
DB_PORT=3306
DB_NAME=edu_live
DB_USER=edu_live
DB_PASSWORD=YourStrongPassword123!

# ===== Redis =====
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

# ===== JWT =====
JWT_SECRET=your_random_secret_key_here_min_32_chars

# ===== ZEGO 即构 =====
ZEGO_APP_ID=your_zego_app_id
ZEGO_SERVER_SECRET=your_zego_secret

# ===== 微信支付 =====
WX_APPID=wx_your_appid
WX_SECRET=your_app_secret
WX_MCHID=your_merchant_id
WX_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----
...
-----END PRIVATE KEY-----
WX_SERIAL_NO=your_serial_no
WX_APIV3_KEY=your_api_v3_key

# ===== 支付宝 =====
ALIPAY_APPID=your_alipay_appid
ALIPAY_PRIVATE_KEY=-----BEGIN RSA PRIVATE KEY-----
...
-----END RSA PRIVATE KEY-----
ALIPAY_PUBLIC_KEY=-----BEGIN PUBLIC KEY-----
...
-----END PUBLIC KEY-----

# ===== 阿里云 OSS =====
OSS_REGION=oss-cn-hangzhou
OSS_ACCESS_KEY_ID=your_key
OSS_ACCESS_KEY_SECRET=your_secret
OSS_BUCKET=your_bucket
```

继续部署：

```bash
# 4. 创建必要目录
mkdir -p logs uploads

# 5. 同步数据库表
npm run db:sync

# 6. 启动服务（PM2）
pm2 start ecosystem.config.js --env production

# 7. 保存 PM2 配置（开机自启）
pm2 save
pm2 startup

# 8. 查看状态
pm2 status
pm2 logs edu-live-server
```

### 3.2 配置 Nginx

复制配置文件：

```bash
sudo cp nginx.conf /etc/nginx/conf.d/edu-live.conf
```

编辑域名和证书路径：

```bash
sudo vim /etc/nginx/conf.d/edu-live.conf
# 修改 server_name 为你的域名
# 修改 ssl_certificate 和 ssl_certificate_key 路径
```

测试并重载配置：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 3.3 配置 HTTPS（SSL 证书）

#### 方式一：使用 Certbot（Let's Encrypt 免费证书）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d api.yourdomain.com

# 自动续期
sudo certbot renew --dry-run
```

#### 方式二：手动上传证书

```bash
# 创建证书目录
sudo mkdir -p /etc/nginx/ssl

# 上传证书文件
sudo cp yourdomain.crt /etc/nginx/ssl/
sudo cp yourdomain.key /etc/nginx/ssl/
sudo chmod 600 /etc/nginx/ssl/*
```

### 3.4 配置防火墙

```bash
# 开放 80 和 443 端口
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable

# CentOS 使用 firewalld
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

## 四、Docker Compose 部署

### 4.1 安装 Docker

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | bash

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

### 4.2 部署步骤

```bash
# 1. 克隆项目
cd /var/www
git clone <your-repo-url> edu-live-server
cd edu-live-server

# 2. 配置环境变量
cp .env.example .env
vim .env  # 填入所有配置

# 3. 启动所有服务
sudo docker-compose up -d

# 4. 查看状态
sudo docker-compose ps

# 5. 查看日志
sudo docker-compose logs -f app
```

### 4.3 常用命令

```bash
# 查看所有容器状态
sudo docker-compose ps

# 查看应用日志
sudo docker-compose logs -f app

# 查看数据库日志
sudo docker-compose logs -f mysql

# 重启应用
sudo docker-compose restart app

# 重新构建并启动
sudo docker-compose up -d --build app

# 进入数据库容器
sudo docker-compose exec mysql mysql -u root -p

# 进入 Redis 容器
sudo docker-compose exec redis redis-cli

# 停止所有服务
sudo docker-compose down

# 停止并删除数据卷（谨慎使用）
sudo docker-compose down -v
```

## 五、第三方服务配置

### 5.1 微信支付配置

1. **申请商户号**: 登录 [微信支付商户平台](https://pay.weixin.qq.com/)
2. **获取 API 证书**: 商户平台 → API 安全 → 申请 API 证书
3. **设置回调地址**: 商户平台 → 产品中心 → 开发配置 → 支付回调 URL
   - 填入: `https://api.yourdomain.com/api/pay/wx/notify`
4. **获取参数**: 
   - AppID（微信公众平台）
   - MCHID（商户号）
   - APIv3 密钥
   - 证书序列号
   - 证书私钥

### 5.2 支付宝配置

1. **创建应用**: 登录 [支付宝开放平台](https://open.alipay.com/)
2. **配置应用**:
   - 应用网关: `https://api.yourdomain.com`
   - 授权回调地址: `https://api.yourdomain.com/api/pay/alipay/return`
3. **设置接口加签方式**: 生成 RSA2 密钥对
4. **开通产品**: 电脑网站支付、手机网站支付
5. **配置异步通知地址**: 
   - 填入: `https://api.yourdomain.com/api/pay/alipay/notify`

### 5.3 ZEGO 即构配置

1. **注册账号**: 登录 [ZEGO 控制台](https://console.zego.im/)
2. **创建项目**: 获取 AppID 和 ServerSecret
3. **配置回调**: 
   - 录制回调地址: `https://api.yourdomain.com/api/zego/callback`
   - 选择回调事件: `RecordFinish`, `StreamStatus`
4. **开通服务**: 实时音视频、云端录制

### 5.4 阿里云 OSS 配置

1. **创建 Bucket**: 登录 [阿里云 OSS 控制台](https://oss.console.aliyun.com/)
2. **配置跨域规则**:
   - 来源: `https://yourdomain.com`
   - 允许 Methods: `GET, POST, PUT`
   - 允许 Headers: `*`
3. **获取 AccessKey**: 阿里云控制台 → AccessKey 管理
4. **配置权限**: Bucket 读写权限设为「私有」或「公共读」

### 7.3 数据库备份

```bash
#!/bin/bash
# 每日自动备份脚本
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/var/backups/edu-live
db_password="YourStrongPassword123!"

mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u root -p$db_password edu_live > $BACKUP_DIR/edu_live_$DATE.sql

# 压缩
gzip $BACKUP_DIR/edu_live_$DATE.sql

# 删除 30 天前的备份
find $BACKUP_DIR -name "edu_live_*.sql.gz" -mtime +30 -delete

echo "备份完成: $BACKUP_DIR/edu_live_$DATE.sql.gz"
```

添加到定时任务：
```bash
crontab -e
# 每天凌晨 3 点备份
0 3 * * * /var/www/edu-live-server/docs/backup.sh >> /var/log/edu-live-backup.log 2>&1
```

## 八、故障排查

### 8.1 服务无法启动

```bash
# 检查环境变量
cat .env | grep -E "^(NODE_ENV|PORT|DB_|REDIS_)"

# 检查端口占用
sudo lsof -i :8080

# 检查数据库连接
mysql -u root -p -e "SHOW DATABASES;"

# 检查 Redis 连接
redis-cli ping

# 查看详细错误
pm2 logs edu-live-server --lines 200
```

### 8.2 支付回调不触发

1. 检查域名是否可外网访问
2. 确认 `.env` 中 `BASE_URL` 配置正确
3. 检查 Nginx access.log 是否有回调请求
4. 微信/支付宝后台确认回调地址配置正确
5. 使用 Postman 模拟回调测试接口是否正常

### 8.3 数据库连接池耗尽

```sql
-- 查看当前连接数
SHOW STATUS LIKE 'Threads_connected';
SHOW STATUS LIKE 'Max_used_connections';

-- 查看最大连接数
SHOW VARIABLES LIKE 'max_connections';

-- 修改最大连接数
SET GLOBAL max_connections = 500;
```

在 `config/database.js` 中调整连接池配置：
```javascript
pool: {
  max: 20,      // 最大连接数
  min: 5,       // 最小连接数
  acquire: 60000,
  idle: 10000
}
```

### 8.4 内存溢出

```bash
# 查看 Node.js 内存使用
pm2 show edu-live-server

# 增加内存限制（package.json）
node --max-old-space-size=2048 app.js

# PM2 配置中设置最大内存
max_memory_restart: '1G'  // 超过 1G 自动重启
```

- [ ] 定期更新系统补丁
- [ ] 定期备份数据库
- [ ] 开启操作日志审计

### 9.2 Nginx 安全配置

```nginx
# 隐藏版本号
server_tokens off;

# 限制请求频率
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
limit_req zone=api burst=20 nodelay;

# 防止 DDOS
limit_conn_zone $binary_remote_addr zone=addr:10m;
limit_conn addr 20;

# 安全响应头
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
```

## 附录

### A. 系统服务脚本

```bash
# /etc/systemd/system/edu-live.service
[Unit]
Description=Edu Live Server
After=network.target mysql.service redis.service

[Service]
Type=forking
User=www-data
WorkingDirectory=/var/www/edu-live-server
ExecStart=/usr/local/bin/pm2 start ecosystem.config.js --env production
ExecReload=/usr/local/bin/pm2 reload ecosystem.config.js --env production
ExecStop=/usr/local/bin/pm2 stop edu-live-server
Restart=always

[Install]
WantedBy=multi-user.target
```

启用：
```bash
sudo systemctl enable edu-live
sudo systemctl start edu-live
sudo systemctl status edu-live
```
