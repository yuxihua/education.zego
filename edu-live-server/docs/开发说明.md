## 一、项目概述

教培直播系统是一套面向教育培训机构的一站式直播解决方案，支持：

- **直播授课**: 基于 ZEGO 即构的实时音视频直播
- **课程管理**: 课程的创建、发布、上下架
- **支付系统**: 微信支付（JSAPI）+ 支付宝（网页/手机支付）
- **学员管理**: 学员注册、报名、作业管理
- **消息推送**: 微信小程序订阅消息推送
- **回放系统**: 直播自动录制并生成回放

## 四、开发环境搭建

### 4.1 环境要求

| 组件 | 版本要求 | 用途 |
|------|---------|------|
| Node.js | >= 18.0.0 | 运行时 |
| MySQL | >= 8.0 | 主数据库 |
| Redis | >= 6.0 | 缓存/会话 |
| npm | >= 9.0.0 | 包管理 |

### 4.2 安装步骤

```bash
# 1. 克隆项目
git clone <repository-url> edu-live-server
cd edu-live-server

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env，填写数据库、Redis、支付等配置

# 4. 创建数据库
mysql -u root -p -e "CREATE DATABASE edu_live CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 5. 同步数据表
npm run db:sync

# 6. 启动开发服务器
npm run dev
```

服务启动后访问: `http://localhost:8080/api/health`

## 五、核心模块说明

### 5.1 认证模块 (`middleware/auth.js` + `routes/auth.js`)

- **JWT Token**: 使用 `jsonwebtoken` 生成和验证
- **Token 刷新**: 支持 AccessToken 自动刷新
- **Token 黑名单**: 用户登出时加入 Redis 黑名单
- **角色权限**: `superadmin` / `admin` / `teacher` / `assistant`

```javascript
// 需要登录
router.get('/profile', auth, ...);

// 需要特定角色
router.post('/course', auth, requireRole(['admin', 'teacher']), ...);

// 可选登录
router.get('/public', optionalAuth, ...);
```

### 5.2 支付模块 (`utils/pay.js` + `routes/pay.js`)

| 功能 | 接口 | 说明 |
|------|------|------|
| 微信下单 | `POST /api/pay/wx/create` | JSAPI 小程序内支付 |
| 微信回调 | `POST /api/pay/wx/notify` | 自动处理支付结果 |
| 支付宝下单 | `POST /api/pay/alipay/create` | 电脑网站支付 |
| 支付宝WAP | `POST /api/pay/alipay/wap/create` | 手机网站支付 |
| 支付宝回调 | `POST /api/pay/alipay/notify` | 异步通知 |
| 订单查询 | `GET /api/pay/order/query` | 查询订单状态 |

**关键逻辑**: 支付回调采用"先返回 200，再异步处理"模式，避免微信/支付宝重复通知。

### 5.3 ZEGO 直播模块 (`routes/zego.js` + `config/zego.js`)

| 功能 | 接口 | 说明 |
|------|------|------|
| 录制回调 | `POST /api/zego/callback` | 处理录制完成事件 |
| 回放查询 | `GET /api/zego/replay/:roomId` | 获取回放信息 |

**录制流程**:
```
直播结束 → ZEGO回调 → 下载MP4 → 转存OSS → 更新回放URL → 通知学员
```

### 5.4 推送模块 (`utils/push.js`)

| 模板 | 函数 | 触发时机 |
|------|------|---------|
| 直播开始提醒 | `notifyLiveStart()` | 讲师点击开播 |
| 购买成功通知 | `notifyPurchaseSuccess()` | 支付回调成功后 |
| 作业批改通知 | `notifyHomeworkGraded()` | 教师批改后 |
| 预约提醒 | `notifyLiveReservation()` | 课程即将开始 |

## 九、常见问题

### Q: 支付回调不触发？

1. 检查 `BASE_URL` 是否为外网可访问的域名
2. 确认微信/支付宝后台配置的回调地址正确
3. 检查 Nginx 日志是否有请求到达
4. 使用 `POST /api/zego/webhook/test` 测试回调接口

### Q: ZEGO 直播无法播放？

1. 检查 `ZEGO_APP_ID` 和 `ZEGO_SERVER_SECRET` 是否正确
2. 确认前端 SDK 初始化参数与后端一致
3. 检查浏览器控制台的网络请求

### Q: 数据库连接失败？

1. 检查 `.env` 中的数据库配置
2. 确认 MySQL 服务已启动
3. 检查数据库用户权限
4. 查看 `logs/` 目录中的错误日志
