# 教培直播系统 - 完整部署指南

> 前后端完整部署，从零到上线

---

## 一、环境要求

| 组件 | 版本 | 用途 |
|------|------|------|
| Node.js | >= 18 | 前后端运行环境 |
| MySQL | >= 8 | 主数据库 |
| Redis | >= 6 | 缓存/会话 |
| npm | >= 9 | 包管理 |

---

## 二、快速开始（5分钟）

```bash
# 1. 解压项目
unzip edu-live-full.zip
cd edu-live-full

# 2. 部署后端
cd edu-live-server
npm install
cp .env.example .env
# 编辑 .env 填入你的数据库/支付/ZEGO 等配置
npm run dev        # 开发模式启动（端口 8080）

# 3. 部署前端（另开终端）
cd ../edu-live-admin
npm install
npm run dev        # 开发模式启动（端口 3000）
```

访问前端：`http://localhost:3000`
访问后端：`http://localhost:8080/api/health`

## 三、生产部署（PM2 方式）

### 3.1 后端部署

```bash
cd edu-live-server

# 安装生产依赖
npm ci --only=production

# 配置环境变量
cp .env.example .env
vim .env   # 填入所有配置项

# 创建必要目录
mkdir -p logs uploads

# 同步数据库
npm run db:sync

# PM2 启动
pm2 start ecosystem.config.js --env production
pm2 save
pm2 startup
```

### 3.2 前端部署

```bash
cd edu-live-admin

# 安装依赖
npm ci --only=production

# 构建生产包
npm run build

# 构建产物在 dist/ 目录，用 Nginx 托管
```

### 3.3 Nginx 配置

```bash
# 复制配置
sudo cp edu-live-server/nginx.conf /etc/nginx/conf.d/edu-live.conf

# 修改配置中的域名和证书路径
sudo vim /etc/nginx/conf.d/edu-live.conf

# 测试并重载
sudo nginx -t
sudo systemctl reload nginx
```

## 四、Docker 部署（推荐）

```bash
cd edu-live-server

# 配置环境变量
cp .env.example .env
vim .env

# 一键启动所有服务（后端 + MySQL + Redis + Nginx）
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f app
```

## 五、清理/重装

```bash
# 停止所有服务
cd edu-live-server
bash cleanup.sh   # 一键清理

# 然后重新按上面步骤部署
```

## 六、关键配置项

编辑 `edu-live-server/.env`：

```bash
# 必填
NODE_ENV=production
PORT=8080
BASE_URL=https://你的域名.com

# 数据库
DB_HOST=localhost
DB_NAME=edu_live
DB_USER=root
DB_PASSWORD=你的密码

# Redis
REDIS_HOST=localhost
REDIS_PASSWORD=你的密码

# JWT 密钥（随机字符串，至少32位）
JWT_SECRET=your_random_secret_key_xxx

# 微信支付
WX_APPID=wx_xxx
WX_MCHID=商户号
WX_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----
...
-----END PRIVATE KEY-----
WX_SERIAL_NO=证书序列号
WX_APIV3_KEY=APIv3密钥

# 支付宝
ALIPAY_APPID=你的APPID
ALIPAY_PRIVATE_KEY=-----BEGIN RSA PRIVATE KEY-----
...
-----END RSA PRIVATE KEY-----
ALIPAY_PUBLIC_KEY=-----BEGIN PUBLIC KEY-----
...
-----END PUBLIC KEY-----

# ZEGO 即构
ZEGO_APP_ID=数字
ZEGO_SERVER_SECRET=你的密钥

# 阿里云 OSS
OSS_REGION=oss-cn-hangzhou
OSS_ACCESS_KEY_ID=你的Key
OSS_ACCESS_KEY_SECRET=你的Secret
OSS_BUCKET=你的Bucket
```

## 七、常见问题

| 问题 | 解决 |
|------|------|
| 端口占用 | `lsof -i :8080` 找到进程并 kill |
| 数据库连接失败 | 检查 .env 中 DB_HOST/DB_PASSWORD |
| 支付回调不触发 | 确认 BASE_URL 是外网可访问的域名 |
| ZEGO 直播失败 | 检查 ZEGO_APP_ID 和 SERVER_SECRET |
| 前端跨域 | 确认 vite.config.js 中 proxy 配置正确 |

### 访问域名显示 JSON：`{"message":"接口不存在"}`

这是 Nginx 将 `/` 也反代到后端导致的。后端只处理 API，请求首页会走 404 返回 JSON。

正确做法：

```nginx
location / {
	try_files $uri $uri/ /index.html;
}

location /api/ {
	proxy_pass http://127.0.0.1:8080;
}
```

如果你使用同域部署（例如 `iot.hualongda.com.cn`），可直接使用：

- `edu-live-server/docs/nginx-iot.hualongda.com.cn.conf`
