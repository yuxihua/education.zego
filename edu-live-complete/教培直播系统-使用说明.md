# 教培直播系统 - 使用说明

## 一、部署步骤

### 1. 一键部署
```bash
# 上传 edu-live-complete.zip 和 deploy-edu-live.sh 到服务器
chmod +x deploy-edu-live.sh
./deploy-edu-live.sh
```

### 2. 手动部署
```bash
# 后端
cd server
npm install
# 修改 .env 配置数据库和 ZEGO
npm start

# 前端
cd edu-live-admin
npm install
npm run build
# dist 目录即为构建产物
```

## 二、老师如何使用（直播推流）

### 首次直播流程
1. **登录后台** - 访问 `http://你的域名`，使用老师账号登录
2. **创建课程** - 进入"课程管理" -> "新建课程"，填写课程信息
3. **创建直播间** - 进入"直播管理" -> "新建直播"，选择刚创建的课程
4. **进入推流页面** - 直播间创建后，点击"开始直播"按钮，或在列表页点击"去推流"
5. **开始推流** - 在推流页面点击"开始直播"，浏览器会请求摄像头权限，允许后即可推流
6. **分享PPT** - 推流页面下方有白板区域，点击"上传课件"可使用 ZEGO 原生功能上传 PPT
7. **结束直播** - 点击"结束直播"按钮，直播正式结束

### 再次进入直播（重新推流）
- 如果直播进行中刷新页面或断线：
  - 重新登录后台 -> "直播管理" -> 找到对应直播间 -> 点击"去推流"
  - 推流页面会自动恢复到之前的直播状态，点击"恢复推流"即可继续
- 如果直播已结束需要重新开播：
  - "直播管理" -> 找到直播间 -> 点击"重新开始"

### 推流页面功能
| 功能 | 说明 |
|------|------|
| 摄像头推流 | 开启摄像头进行视频直播 |
| 屏幕分享 | 分享电脑屏幕（适合演示软件操作） |
| ZEGO白板 | 使用 ZEGO 原生 SuperBoard 进行板书和PPT讲解 |
| 上传课件 | 调用 ZEGO 原生 `uploadFile()` 上传 PPT/PDF，**非自定义axios上传** |
| 学生互动 | 实时查看学生聊天消息 |
| 全员禁言 | 一键控制学生发言权限 |
| 在线人数 | 实时显示当前观看学生数量 |

## 三、学生如何观看直播

### 观看方式
学生无需登录，直接通过链接进入：
```
http://你的域名/watch/{直播间ID}
```

例如：`http://education.zego/watch/1`

### 进入直播间后的功能
| 功能 | 说明 |
|------|------|
| 观看视频 | 自动播放老师的摄像头画面 |
| 屏幕分享 | 老师分享屏幕时自动显示 |
| 白板课件 | 同步显示老师的 PPT/白板内容，支持翻页 |
| 课堂聊天 | 右侧聊天区可发送消息与老师互动 |
| 在线人数 | 显示当前在线同学数量 |

### 重新进入直播
- **直播中刷新页面**：自动保持身份重新进入同一房间，聊天记录保留
- **断线重连**：页面会自动检测并重新连接
- **切换标签页后回来**：自动恢复连接状态
- **直播结束后再次打开链接**：显示"直播已结束"状态

## 四、直播状态说明

| 状态 | 说明 |
|------|------|
| 即将开始 | 老师已创建直播间但尚未开始推流 |
| 直播中 | 老师正在进行推流，学生可实时观看 |
| 暂停 | 老师暂停了直播（临时中断） |
| 已结束 | 直播正常结束，学生进入显示回放或结束提示 |
| 已关闭 | 直播间被管理员关闭 |

## 五、PPT/课件上传说明

本系统**使用 ZEGO 原生白板功能**上传和展示 PPT：

1. 老师在推流页面点击"上传课件"按钮
2. 调用 ZEGO SuperBoard 的 `uploadFile()` 方法原生上传
3. 上传完成后自动创建白板视图
4. 学生端通过 ZEGO SuperBoard SDK 同步显示

**注意**：不需要自定义 axios 上传接口，完全由 ZEGO 白板组件处理。

## 六、常见问题

### Q: 学生页面黑屏/看不到视频？
A: 检查以下几点：
- 老师是否已经开始推流（状态是否为"直播中"）
- 浏览器是否允许摄像头/麦克风权限
- ZEGO APP ID 和 Server Secret 是否配置正确
- 打开浏览器开发者工具(F12)查看控制台报错

### Q: 刷新页面后身份丢失？
A: 系统已实现身份持久化：
- 学生：localStorage 保存 userID，刷新后自动恢复
- 老师：localStorage 保存 token，刷新后自动重新获取用户信息

### Q: 白板不显示/课件看不到？
A: 
- 确认老师已上传课件
- 检查 ZEGO SuperBoard SDK 是否加载成功
- 确认 ZEGO 配置中的 appID 正确

### Q: 聊天消息发送失败？
A:
- 检查是否被老师禁言
- 检查网络连接是否正常
- 查看浏览器控制台是否有 API 报错

## 七、API 接口速查

| 接口 | 说明 |
|------|------|
| GET /api/health | 健康检查 |
| POST /api/auth/login | 登录 |
| GET /api/auth/me | 获取当前用户 |
| GET /api/live/rooms | 直播间列表 |
| GET /api/live/room/:id | 直播间详情 |
| POST /api/live/room | 创建直播间 |
| POST /api/live/room/:id/start | 开始直播 |
| POST /api/live/room/:id/stop | 结束直播 |
| GET /api/live/token | 获取 ZEGO Token |
| POST /api/live/room/:id/online | 上报在线 |
| POST /api/live/room/:id/heartbeat | 心跳保活 |
| GET /api/live/room/:id/chat/history | 聊天历史 |

## 八、技术栈

- **前端**：Vue 3 + Vite + Element Plus + Pinia
- **后端**：Node.js 18+ + Express 4 + Sequelize ORM
- **数据库**：MySQL 8.0
- **缓存**：Redis 6
- **直播**：ZEGO Express Engine WebRTC + SuperBoard
- **部署**：Nginx + PM2
